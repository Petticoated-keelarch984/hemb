- Fastest bearer: tcp_0 (~200ms)

## Notes
1. cellular_0 was configured as a bearer in bond_mega_01 but the HeMB bonder only used 2 bearers (mesh_0 + tcp_0). The cellular sendFnProvider is not wired for HeMB symbol transmission — only for app-level SMS. This is expected: cellular/SMS is a message-level transport, not a packet-level bearer for RLNC symbols.
2. IMT satellite send failed with status "dead" (delivery #13). Signal was 2 bars (fair) but the MO send timed out. Single attempt only per test constraints — no retry.
3. The bond_live_01 queue on mule01 is at depth 500 (full). This doesn't affect the mega test but indicates background HeMB traffic is saturating the queue.
4. The mesh_0 relay (delivery #15, status=sent) was dispatched via LoRa but mule01 message DB does not yet show the relayed payload. The Reticulum routing may require the destination hash to be specified in the access rule's forward_options.
